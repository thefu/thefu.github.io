Rust中的集合Collection

发表于 2023-09-26

集合（Collections）是存储多个值的数据结构，主要存放在标准库 std::collections 模块中。集合的大小可以动态变化，且数据通常存储在堆上。

Rust 标准库中的主要集合类型

集合类型 说明 特点
Vec<T> 动态数组 元素有序、可变长、随机访问高效
VecDeque<T> 双端队列 支持两端高效插入/删除
LinkedList<T> 双向链表 插入删除快，随机访问慢
HashMap<K,V> 哈希表 基于哈希的键值存储，无序
BTreeMap<K,V> 平衡树映射 基于排序的键值存储，有序
HashSet<T> 哈希集合 不重复元素，无序
BTreeSet<T> 有序集合 不重复元素，有序
BinaryHeap<T> 堆结构 可用于优先队列（最大堆）

1. 动态数组 Vec<T>

Vec<T> 是 Rust 中最常用的集合，本质是动态数组，支持在末尾高效插入/删除元素，且可通过索引随机访问。

2. 双端队列 VecDeque<T>

VecDeque<T>（Double-ended Queue）是双端队列，底层使用环形缓冲区实现。支持在首尾两端高效插入/删除元素。

3. 双向链表 LinkedList<T>

LinkedList<T> 是双向链表，元素通过指针连接，不连续存储。

性能考量

  • 实际性能较差：因元素不连续，缓存友好性低，随机访问效率低（O(n)）
  • 极少使用：除非明确需要链表的理论特性，否则优先用 Vec 或 VecDeque

基本操作

常用方法

方法 说明 返回类型

push_back(value) 尾部插入 ()
push_front(value) 头部插入 ()

front() / back() 获取首尾引用 Option<&T>
len() / is_empty() 获取长度/是否为空 usize / bool
clear() 清空链表 ()
append(&mut other) 拼接另一个链表 ()
iter() / iter_mut() 遍历 迭代器

4. 哈希表 HashMap<K,V>

HashMap<K, V> 基于哈希表实现，通过哈希函数将键映射到存储位置，查询和插入的平均时间复杂度为 O(1)。

特点

  • 无序存储（键的顺序不固定）
  • 键 K 需实现 Hash 和 Eq trait（用于哈希计算和相等性比较）
  • 性能依赖哈希函数的质量（避免哈希冲突）

创建方式

常用方法

方法 说明 返回类型

with_capacity(n) 预分配容量 HashMap<K,V>

remove(key) 删除键 Option<V>
get(key) 获取值引用 Option<&V>
get_mut(key) 可变引用 Option<&mut V>
contains_key(key) 是否存在 bool
entry(key) 获取或创建 entry Entry<K,V>
len() / is_empty() 长度/是否空 usize / bool
keys() / values() / iter() 迭代键、值、键值对 迭代器
clear() 清空 ()

HashMap 示例

5. 有序映射 BTreeMap<K,V>

BTreeMap<K, V> 基于 B 树实现，键值对按键的排序顺序存储，查询时间复杂度为 O(log n)。

特点

  • 有序存储：键值对按键的自然顺序（或自定义排序）有序存储
  • 键 K 需实现 Ord trait（用于排序）
  • 支持范围查询：如获取”大于 x 且小于 y”的键值对

基本操作

contains_key(key) 检查存在 bool
len() / is_empty() 长度/空 usize / bool
iter() 遍历有序键值 迭代器
range(range) 范围遍历 迭代器
first_key_value() / last_key_value() 最小/最大键 Option<(&K,&V)>
clear() 清空 ()

BTreeMap 示例

6. 哈希集合 HashSet<T>

HashSet<T> 基于 HashMap<T, ()> 实现，存储唯一元素，无序。

特点

  • 元素唯一：插入重复元素会被忽略
  • 元素 T 需实现 Hash 和 Eq trait
  • 插入、查询、删除的平均时间复杂度为 O(1)

基本操作

7. 有序集合 BTreeSet<T>

BTreeSet<T> 基于 BTreeMap<T, ()> 实现，存储唯一元素，且按元素顺序排序。

特点

  • 元素唯一且有序：按 Ord trait 排序
  • 元素 T 需实现 Ord trait
  • 支持范围查询，查询时间复杂度为 O(log n)

集合选择指南

场景 推荐集合 理由
通用有序集合 Vec<T> 简单高效，大多数场景首选
双端队列（FIFO/LIFO） VecDeque<T> 两端高效操作
键值存储（无序） HashMap<K,V> 平均 O(1) 查询
键值存储（有序） BTreeMap<K,V> 支持范围查询，O(log n) 查询
去重集合（无序） HashSet<T> 快速去重和存在性检查
去重集合（有序） BTreeSet<T> 支持范围查询，有序遍历
优先队列 BinaryHeap<T> 高效取最大/最小值
链表操作 LinkedList<T> 极少使用，除非明确需要

总结

  • Vec 是最常用的集合，适合大多数场景
  • HashMap/BTreeMap 用于键值存储，选择取决于是否需要有序性
  • HashSet/BTreeSet 用于去重和集合操作
  • VecDeque 适合双端队列操作
  • BinaryHeap 实现优先队列功能
  • LinkedList 应避免使用，实际性能不如 Vec/VecDeque
