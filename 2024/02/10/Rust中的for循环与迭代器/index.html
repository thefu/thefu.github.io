Rust中的for循环与迭代器

发表于 2024-02-10 更新于 2025-11-26

for 循环是迭代器的语法糖，而迭代器是 Rust 处理集合（如 Vec、HashMap）的核心抽象。

1. for 循环

for 循环本质

for 循环本质是自动遍历一个”可迭代对象”（实现 IntoIterator trait 的类型），底层逻辑如下：

  1. 对目标对象调用 into_iter() 方法，将其转换为迭代器（Iterator）
  2. 循环调用迭代器的 next() 方法，每次获取一个元素（Some(T)）
  3. 当 next() 返回 None 时，循环终止

for 循环的用法

任何实现 IntoIterator 的类型，都能直接用 for 循环遍历。

1. 遍历集合

需注意所有权与借用模式，三种核心模式：

  • 所有权转移遍历：into_iter()

    • 转移所有权，T 类型
    • for x in v

  • 不可变借用遍历：iter()

    • 不可变引用 &T，需通过解引用（*）访问元素值
    • for x in &v

  • 可变借用遍历：iter_mut()

    • 可变引用 &mut T，需通过解引用（*）修改元素
    • for x in &mut v

2. 计数循环 - Range 区间遍历

3. 键值对遍历（HashMap/BTreeMap）

4. 遍历索引与值

三种遍历模式对比

for 循环形式 底层调用 元素类型 说明
for x in v v.into_iter() T（原值） 所有权转移（消耗集合）
for x in &v v.iter() &T（不可变引用） 不可变借用（只读集合）
for x in &mut v v.iter_mut() &mut T（可变引用） 可变借用（修改集合）
for i in 0..n Range 迭代器 值拷贝 计数循环

绑定解构

迭代器遍历中元素的类型由 Iterator trait 中的 Item 确定。对于引用类型，可通过 &（不是取地址）进行解构。以 iter()（Item 是 &T）遍历为例：

  • **用 x**：绑定 &T（保留引用），需通过 *x 解引用操作值
  • **用 &x**：解构一层引用，直接绑定 T（原值），无需解引用

2. 迭代器

Rust 的迭代器是实现了 Iterator trait 的对象。迭代器不仅仅是一个”遍历工具”，还能惰性处理数据流，即惰性求值（lazy evaluation）。

迭代器的特点

多数方法不会立即执行，只是返回一个新的迭代器。直到调用 .collect()、.for_each() 等最终消费方法时，才计算。

Iterator trait

适配器（Adapter）

适配器是返回新迭代器的方法，用于转换、筛选、组合原迭代器。多个适配器可组成”链式”，即通过把多个迭代器适配器组合在一起形成流水线式的数据处理流程。

常见适配器

适配器 说明
map(f) 对元素应用 f，转换元素类型
filter(f) 保留满足条件 f 的元素，返回 bool
flat_map(f) 对元素应用 f，扁平化结果
rev() 反向遍历（需迭代器实现 DoubleEndedIterator）
skip(n) 跳过前 n 个元素
take(n) 取前 n 个元素，之后终止迭代
chain(iter2) 拼接两个迭代器
enumerate() 同时获取元素的索引和值，返回一个元组 (usize, T)

适配器链式处理示例

消耗性方法（Consuming Adapter）

消耗型方法会遍历迭代器、消耗元素，并返回最终结果（触发惰性求值）。

常见消耗性方法

方法 说明
collect<T>() 收集元素为集合 T
count() 返回元素个数
fold(init, f) 折叠元素（聚合计算），init 为初始值
any(f) 存在元素满足 f，则返回 true
all(f) 所有元素满足 f，则返回 true
find(f) 查找第一个满足 f 的元素，返回 Option<Item>
for_each(f) 对每个元素执行 f，等价于 for 循环

消耗性方法示例

无限迭代器

无限迭代器（如 iter::repeat、iter::cycle），需通过 take(n) 等方法限制迭代次数，否则会陷入死循环。

常见无限迭代器

  • **iter::repeat(val)**：无限重复 val
  • **iter::cycle()**：循环迭代原迭代器（原迭代器需是 Clone）
  • **(0..).step_by(n)**：无限递增序列（步长 n）

自定义迭代器

要创建自定义迭代器，只需实现 Iterator trait 的 next() 方法（其他方法为默认实现）。

Iterator trait 继承了 IntoIterator trait 的默认实现（IntoIterator::into_iter() 会直接返回迭代器本身（Self））。因此，只要一个类型实现了 Iterator::next()，就自动支持 for 循环遍历。

自定义容器

容器要被 for 循环遍历，需让容器（或其引用）实现 IntoIterator trait：

  • iter() -> 返回不可变引用的迭代器
  • iter_mut() -> 返回可变引用的迭代器
  • into_iter() -> 转移所有权迭代器

总结

for 循环

  • for 循环是 IntoIterator trait 的语法糖
  • 三种核心模式：所有权转移（into_iter()）、不可变借用（iter()）、可变借用（iter_mut()）
  • 支持 Range、键值对、索引+值 等多种遍历方式

迭代器

  • 惰性求值：适配器不会立即执行，只有消耗型方法才会触发计算
  • 适配器链式处理：通过组合多个适配器实现复杂的数据处理流程
  • 自定义迭代器：实现 Iterator trait 的 next() 方法即可
  • 自定义容器：实现 IntoIterator trait 支持 for 循环遍历
  • 无限迭代器：需要通过 take() 等方法限制迭代次数

迭代器是 Rust 处理集合和序列的核心工具，具有高效、安全、表达力强的特点。
