Rust中的Enum与Struct详解

发表于 2024-02-13

在 Rust 中，struct（结构体）和 enum（枚举）是两种核心的自定义类型，分别用于组合相关数据和表示互斥的可能性。

结构体（struct）

struct 用于将多个相关的值组合在一起，形成一个有意义的组。

具名结构体（named struct）

最常用的结构体形式，字段有明确的名称，可读性强，适合数据含义明确的场景。

  • 字段默认私有（仅当前模块内可见）
  • 通过 impl 实现关联函数（方法）
  • 通过 .. 实现赋值（所有未指定的字段，全部替换为 .. 对应变量中的）；若字段为移动语义，所有权被移走
  • 通过 trait 实现多态

基本示例

可见性

默认为私有，可通过 pub 修饰符修改：

修饰符 作用域 典型场景
pub 全局可见（跨 crate） 完全公开
pub(crate) 整个当前 crate 内可见 库内部共享
pub(super) 父模块可见 子模块辅助函数
pub(in path) 指定模块路径内可见 最灵活的控制
（无修饰） 当前模块私有 默认私有

Trait（继承/多态）

Rust 没有传统 OOP 的类继承（没有基类/子类）。用 trait + impl 来实现接口/多态。

Trait 定义方法签名集合，实现 trait 的类必须提供所有对应方法：

  • 静态分发（T: Trait）在编译期确定具体类型（单态化，性能好）。
  • 动态分发（&dyn Trait 或 Box<dyn Trait>）通过虚函数表 (vtable)，在运行时分发（适合异构集合/插件式架构）。

泛型与生命周期

结构体可以是泛型的，也可以带生命周期参数。

泛型说明：

语法 说明
struct Point<T> 定义泛型结构体
impl<T> Point<T> 为所有类型实现方法
impl Point<i32> 为特定类型实现方法
T: Display 泛型约束，要求类型实现 Display trait
where T: Clone, U: Debug 泛型约束，更清晰的约束语法

元组结构体（tuple struct）

字段没有名称，仅通过位置区分，适合数据结构简单、字段含义可通过位置推断的场景。

单元结构体（unit struct）

一种不包含任何字段的结构体。它的定义形式类似于一个”空”的结构体：

  • 在内存中不占用任何空间（size_of::<UnitStruct>() == 0）
  • 主要用于标记类型（表示”存在性”而非”数据”）

枚举（enum）

枚举的核心作用是定义一个类型，其值是有限的、互斥的几种可能变体（variants）之一。每个变体可以关联不同类型和数量的数据。

枚举通过 enum 关键字定义，每个变体可以是”空”的，也可以携带数据（类似元组或结构体）。

基本定义

模式匹配

枚举的核心用法是通过 match 表达式匹配其变体，从而处理不同情况（类似”多分支条件判断”，但更安全）。

常见枚举

Option<T> - 表示可选值

Option<T> 是 Rust 中用于表示”有值”或”无值”的枚举，彻底避免了空指针（Null）问题。

Result<T, E> - 表示操作结果

Result<T, E> 用于表示操作”成功”或”失败”的结果，是 Rust 错误处理的核心类型。

Ordering - 表示比较结果

Ordering 用于表示两个值的比较结果（小于、等于、大于）。

Struct vs Enum 对比

特性 Struct Enum
用途 组合相关数据（”与”关系） 表示互斥的可能性（”或”关系）
字段 一个实例包含所有字段 一个实例只属于某个变体
内存 所有字段连续存储 只存储当前变体的数据
匹配 直接访问字段 必须通过模式匹配
多态 通过 trait 实现 通过模式匹配实现
适用场景 数据聚合、数据模型 有限状态、错误处理、可选值

总结

  • Struct：用于组织相关数据，所有字段同时存在，适合表示复合对象
  • Enum：用于表示多个互斥状态，任一时刻只有一个变体，适合表示有限的可能性
  • Trait：Rust 的多态机制，通过静态分发（性能好）和动态分发（灵活）提供灵活的抽象
  • Pattern Matching：枚举的强大特性，在编译时保证所有情况都被处理
