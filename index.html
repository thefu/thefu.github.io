Harness Engineering到底是什么

发表于 2026-05-28

最近 AI 工程圈里，一个新词开始频繁出现：Harness Engineering。它出现在 OpenAI 的工程实践文章里，也出现在 Anthropic 关于 long-running agents 的实践总结里，Martin Fowler 网站也专门写过第一印象。这个概念之所以引发讨论，是因为它试图回答一个很现实的问题：当大模型已经足够强之后，工程师到底应该继续优化 Prompt，还是应该把精力放在模型外部的系统设计上？

这期视频的核心观点可以概括为一句话：Harness Engineering 不是简单的新名词炒作，它是一种围绕 Agent 搭建运行系统的工程思维；但它也未必是最终形态，因为模型能力越强，很多外部约束和补救机制就越可能被模型自身吸收。

Go 语言中的 goroutine

goroutine 是 Go 语言并发编程中最核心的概念。它可以理解为由 Go 运行时管理的轻量级执行单元，写法上像启动一个函数，运行时则由调度器负责把大量 goroutine 分配到少量操作系统线程上执行。

相比直接使用线程，goroutine 的创建成本更低，栈空间可以按需增长，适合用来处理网络请求、后台任务、并发 I/O、流水线计算等场景。

Go 语言中的 channel

channel 是 Go 语言中用于 goroutine 之间通信的核心机制。Go 并发编程有一句很经典的话：不要通过共享内存来通信，而要通过通信来共享内存。channel 正是这句话在语言层面的体现。

通过 channel，不同 goroutine 可以安全地发送和接收数据，避免直接共享变量带来的数据竞争问题。
